# DTN 节点化架构 - 发展历程

## 项目简介

DTN（Discrete Trigger Network，离散触发网络）节点化架构是一个革命性的 AI 系统，采用插件式架构，所有功能都由独立的节点实现。

**核心理念：绕开大模型、绕开权重，让系统自己长出知识。**

### 🎯 架构发现：节点化革命

**时间：** 架构重构阶段

**核心发现：**
> "所有的机制与各种规则各种功能都是一个一个一个的节点吗？全部搞成技能与节点怎么样？这样的话dtn或者只需要一个搞出来一个框架就行了？"

**这个发现改变了整个项目方向！**

**新架构设计：**
```
DTN 框架（只负责调度）
├── 节点注册表
├── 节点调度器
└── 各种独立节点
```

**优势：**
- 插件式架构
- 无限扩展
- 易于优化
- 易于测试

### ⭐ v5.0 - 重构：第一性原理

**时间：** 最终重构阶段

**核心思想：**
> "最高级的第一性原理，最高级的逆向工程，高效，极简，夯实基础，体检健康度"

**第一性原理分析：**

1. **什么是节点？** - 一个可执行单元
2. **节点有哪些类型？** - 技能、规则、数据、功能
3. **如何管理节点？** - 注册表 + 调度器
4. **如何扩展？** - 继承基类，实现 execute

**极简设计：**
```
4 个基类：
  ├── Node (基类)
  ├── SkillNode (技能节点)
  ├── RuleNode (规则节点)
  ├── DataNode (数据节点)
  └── FuncNode (功能节点)

2 个管理器：
  ├── NodeRegistry (注册表)
  └── NodeScheduler (调度器)

1 个原则：
  └── 继承基类，实现 execute
```

**主要成果：**
- 创建 `framework/core.py` - 核心基类
- 创建 `framework/registry.py` - 注册表和调度器
- 迁移所有节点到新架构
- 创建完整测试和演示

## 设计原则

1. **第一性原理** - 从最基本的问题出发
2. **极简设计** - 最少代码最多功能
3. **清晰命名** - 节点类型明确
4. **统一管理** - 注册表 + 调度器
5. **易于扩展** - 继承基类即可
